Cargo Tracker

Applied Domain-Driven Design Blueprints for Jakarta EE

Based on the sample application from Eric Evans' Domain-Driven Design book, Cargo Tracker demonstrates how to apply DDD principles — entities, value objects, aggregates, repositories, domain services, and domain events — in a modern Jakarta EE context.

Explore the Application

Three interfaces to interact with the cargo tracking system

Public Tracking

Track any cargo in real time using its tracking ID. View the full itinerary, handling history, and current transport status — no login required.

Track Cargo

Administration

Manage the entire cargo lifecycle. Register new cargo, book routes and voyages, assign handling events, and view the full cargo manifest.

Open Dashboard

Mobile Event Logger

Log handling events on the go. Designed for port workers and customs agents to record loading, unloading, and customs clearance events from any device.

Open Logger

Client Applications

Companion apps that connect to the Cargo Tracker REST API, ported from the original DDDSample regapp to modern stacks

JavaFX Client

A rich desktop client built with JavaFX and CDI/Weld. Provides a full-featured registration interface for managing cargo bookings and tracking operations.
